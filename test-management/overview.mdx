# Test Management

<img src="/images/test-management.png" alt="Collabase Test Management" style={{ borderRadius: '8px', marginBottom: '24px' }} />

Test Management is the quality operating system built into every Collabase Space. It gives QA teams and developers a structured place to define what needs to be verified, execute it, and measure quality over time — all without leaving Collabase.

## How it works

<CardGroup cols={2}>
  <Card title="Test Cases" icon="file-lines" href="/test-management/test-cases">
    Define the steps, preconditions, and expected results for each thing to verify. Organize cases into suites by feature or component.
  </Card>
  <Card title="Test Runs" icon="play" href="/test-management/test-runs">
    Execute all READY cases in a single run session. Each result is recorded independently — pass, fail, blocked, or skipped.
  </Card>
  <Card title="Milestones" icon="flag" href="/test-management/milestones">
    Group runs under a release or sprint milestone to get an aggregate quality view across all linked runs.
  </Card>
  <Card title="Suites" icon="folder" href="/test-management/suites">
    Group test cases logically by feature, module, or priority to keep large projects organized.
  </Card>
</CardGroup>

## Quality workflow

```
Define Cases → Mark READY → Create Run → Execute & Record → Attach to Milestone → Ship
```

1. **Define** test cases for every user story or feature
2. **Review and mark READY** — only verified cases are included in runs
3. **Create a test run** — Collabase enrolls all READY cases automatically
4. **Execute** — testers record PASS/FAIL/BLOCKED/SKIP for each case
5. **Attach to milestone** — link to the release for an aggregate quality snapshot
6. **Ship** — use milestone pass rates as your quality gate

## Automation integration

Test Management integrates directly with the Automation engine:

| Event | Trigger available |
|---|---|
| Test case fails | ✅ `collabase.testCase.failed` |
| Test case passes | ✅ `collabase.testCase.passed` |
| Test run started | ✅ `collabase.testRun.started` |
| Test suite created | ✅ `collabase.testSuite.created` |

Use these to automatically notify Slack, create Jira issues for failures, or trigger CI pipelines on test completion.

## Roles

| Role | Can do |
|---|---|
| **Space Owner / Admin** | Create projects, manage members, delete |
| **Member** | Create cases, execute runs, record results |
| **Viewer** | View cases, runs, and results (read-only) |

## Admin configuration

Admins can configure access to Test Management per Space under **Admin → Apps**. There are no instance-wide settings specific to Test Management beyond Space permissions.
