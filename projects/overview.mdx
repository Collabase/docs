# Projects

<img src="/images/projects.png" alt="Collabase Projects board view" style={{ borderRadius: '8px', marginBottom: '24px' }} />

Projects is the task and project management app built into every Collabase Space. It combines a flexible kanban board, sprint planning, queue-based helpdesk workflows, SLA tracking, and a full task history — all connected to Docs, Test Management, Automation, and the Registry.

## How Projects is organized

Every Space can have multiple Projects. Each Project is a self-contained workspace with its own workflow, task types, members, and settings.

## Views

Projects supports multiple task views. Switch between them from the view toggle in the top-right of the board.

| View | Best for |
|---|---|
| **Board** | Kanban-style drag-and-drop by status column |
| **List** | Dense sortable table of all tasks |
| **Sprint** | Sprint backlog and active sprint tracking |
| **Queue** | Triage inbox organized by queue |
| **Timeline** | Gantt-style view of tasks with due dates |

## Roles and access

Project access is controlled per project, independent of Space membership.

| Role | Permissions |
|---|---|
| **Owner** | Full project configuration, archive, delete |
| **Member** | Create, assign, and resolve tasks |
| **Viewer** | Read-only access to tasks and boards |

Manage members from **Project → Settings → Members**.

## Admin configuration

Admins can configure global task types, custom fields, and worklog attributes that apply across all projects in the instance. See **Admin → Task Types** and **Admin → Custom Fields**.
