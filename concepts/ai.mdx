Collabase integrates AI throughout the platform under the name **Collabase Brain**. All AI features share a single provider configuration set by your administrator. Configure the provider once and every feature benefits automatically.

AI features are **off by default**. An admin must enable them under **Admin → Settings → AI** before they are available to users. See [AI Configuration](/admin/ai-configuration) for setup instructions.

## Writing assistant

The AI writing assistant is available anywhere you write content in Collabase. Type `/ai` on any line to open the action menu.

| Command | What it does |
|---|---|
| **Generate** | Write new content from a prompt you type |
| **Summarize** | Condense the current page or selected text into a brief summary |
| **Improve writing** | Refine grammar, clarity, and overall tone |
| **Fix spelling & grammar** | Correct errors only — structure and tone are preserved |
| **Make shorter** | Trim the selected text to its essential points |
| **Make longer** | Expand the selection with more detail |
| **Continue writing** | Extend the text from the current cursor position |
| **Translate** | Translate selected text to a language you choose |

If text is selected, the command operates on the selection. If nothing is selected, it operates on the paragraph at the cursor.

## Semantic search

When semantic search is enabled by your admin, you can ask questions in plain language using the Brain chat or the global search bar. Collabase retrieves relevant content from your Docs pages and generates a grounded answer with links to the source material.

**Example questions:**
- "What is our approval process for purchasing new software?"
- "Which team is responsible for infrastructure security?"
- "What were the key decisions from the Q2 planning meeting?"

<Note>
  Semantic search covers Docs pages. Tasks, registry objects, and test cases are matched by keyword search.
</Note>

Pages are indexed automatically when they are saved. New content is available in semantic search within a few seconds.

## AI in Automation

The **AI / LLM** automation connector lets you run AI actions inside any automation pipeline. Use it to:

- Generate a summary of a failed test run before creating a bug report
- Classify incoming support requests and route them to the right project
- Draft a task description based on data from another system

AI nodes connect to the same provider configured globally, or to a separately configured connection if you need a different model for automation use cases. See the Automation documentation for details on setting up the AI connector and chaining it with other nodes.

## AI provider options

Your administrator chooses which AI provider Collabase uses. The choice affects privacy, cost, and capability.

### Local AI (Ollama) — privacy-first

| Aspect | Detail |
|---|---|
| **Where processing happens** | On your own server |
| **Data residency** | Nothing leaves your infrastructure |
| **Internet required** | No — works fully offline after setup |
| **Cost** | No per-token fees — only your own server hardware |
| **Setup** | Requires installing Ollama on your server and downloading a model |

Ollama is the recommended choice for organizations that cannot allow content to be sent to external services — such as organizations handling sensitive legal, financial, or healthcare data.

### Cloud AI providers

| Provider | Notes |
|---|---|
| **OpenAI** (GPT-4o, GPT-4o-mini) | Widely used, strong general-purpose models |
| **Anthropic** (Claude) | Strong at writing, summarization, and following instructions |
| **Groq** | Fast inference, generous free tier |
| **Google Gemini** | Large context window, multimodal capability |
| **Azure AI Foundry** | Enterprise OpenAI models in your Azure subscription — useful for organizations already in Azure |

Cloud providers offer more capable models and require no server hardware, but content sent to AI actions is processed on the provider's infrastructure. Review each provider's data processing terms if your organization has specific requirements.

## Which features work with local vs. cloud AI

| Feature | Works with Ollama | Works with cloud providers |
|---|---|---|
| Writing assistant (/ai command) | Yes | Yes |
| Task description generation | Yes | Yes |
| Automation AI nodes | Yes | Yes |
| Semantic search (keyword) | Yes — no AI needed | Yes — no AI needed |
| Semantic search (RAG / plain-language questions) | Yes, with an embedding model (e.g. `nomic-embed-text`) | Yes |

## AI logs

Admins can see a complete history of every AI request made across the platform at **Admin → Settings → AI → Logs**. Each log entry shows:

| Field | Description |
|---|---|
| **Timestamp** | When the request was made |
| **Feature** | Which feature triggered it — editor, semantic search, or automation |
| **Model** | The model that processed the request |
| **Tokens used** | Total token count for the request and response |
| **Latency** | Time from request to response |
| **Status** | Success, or an error message |

Use the logs to debug AI issues — if the writing assistant is not working for a user, the error in the log tells you exactly what went wrong.

### Common errors and fixes

| Error message | What it means | Fix |
|---|---|---|
| `Connection refused` | Ollama is not running or not reachable | Start Ollama on the server |
| `model not found` | The model has not been downloaded to Ollama | Run `ollama pull <model-name>` on the server |
| `invalid_api_key` | The API key for the cloud provider is wrong or expired | Generate a new key from the provider and update it in **Admin → Settings → AI** |
| `context length exceeded` | The input is too long for the model | Reduce the input size or switch to a model with a larger context window |

## For administrators

- AI setup: **Admin → Settings → AI** — see [AI Configuration](/admin/ai-configuration)
- Space-level model override: **Space Settings → AI** — lets individual spaces use a different model than the default
- AI logs: **Admin → Settings → AI → Logs** — shows every AI request, tokens used, latency, and errors
