# API Reference

The Collabase REST API lets you integrate Collabase with external tools, build custom automations, and manage resources programmatically.

## Base URL

```
https://your-collabase.example.com/api
```

All `v1` routes are stable. Routes without a version prefix are internal BFF routes used by the Collabase web application.

## Authentication

All API requests require a session token. Authenticate by passing your API key as a Bearer token:

```bash
Authorization: Bearer cba_live_xYz123ABCD...
```

Use your Collabase session token from the developer tools → Application → Cookies → `next-auth.session-token`, or generate a token via the Admin API (coming soon).

## Response format

All responses return JSON. Successful responses use `2xx` status codes. Errors return a JSON object with an `error` field.

```json
{ "error": "Unauthorized" }
```

## Rate limiting

The API is rate-limited to 300 requests per minute per authenticated user. Exceeding this returns `429 Too Many Requests`.
